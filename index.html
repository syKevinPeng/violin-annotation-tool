How to Annotate

Workflow Overview

  1. Load files — student audio (WAV/MP3/OGG/FLAC) + score (MusicXML).
  2. Listen and mark errors — click an error type (or press 1–9), then click the affected note(s) in the score. A popup will ask for severity and any extra detail.
  3. Run Tap-Align — click 🎵 Tap-Align in the toolbar. Audio plays from the start; press Space on every note onset you hear. The tool uses your error annotations to map taps to score notes automatically.
  4. Export Aligned JSON — gives you a per-note record with audio onset times and status labels, ready for downstream model training/eval.

Tap-Align Mode

Click 🎵 Tap-Align in the toolbar after marking all errors. Audio starts from the beginning and the next-expected score note is highlighted in blue.

Tap rule: tap Space on every note onset you hear, including wrong notes (still tap them — you're recording the timing, not judging correctness). The only exception is skipped notes: don't tap, the tool already knows to omit them.

Why errors first? The error labels tell the tool how to map your taps to score notes when the performance deviates structurally (skips, repeats, swaps). Without them, tap-align can't know where in the score each tap belongs.

KeyAction
SpaceRecord a tap at the current playhead time
BackspaceUndo the most recent tap
PPlay / pause audio (Space is reassigned to Tap during this mode)
EscEnd Tap-Align (saves your taps)

When you finish, the tool reports any mismatch between recorded and expected tap counts. A mismatch usually means a missing error annotation or a missed tap.

Repetition: Clean vs Corrective

When you mark a repetition, the tool asks why the passage was repeated. The answer changes how labels are assigned across passes:

  • Clean repeat — the first pass was fine; the extra play is the error. Pass 1 keeps any underlying labels; pass 2+ becomes repetition.
  • Corrective repeat — the first pass had a mistake, the last pass is the corrected take. Pass 1 (and any middle passes) keep the underlying error label; the final pass becomes clean.

If a corrective repeat fixed an underlying error (e.g., a wrong note), mark that error separately as a normal annotation at its score location. The tool composes them automatically.

Export Formats

  • Export CSV — flat spreadsheet of error annotations only (no per-note onsets). Opens in Excel / Google Sheets.
  • Export JSON — full session state: annotations + raw taps + assigned per-note onsets. Re-importable into the tool.
  • Export Aligned JSON — per-score-note records with passes: [{audio_t, pass_number, status}]. This is the format downstream model training and evaluation expects.

Tips

  • Listen at normal speed first to get an overview, then at 0.5× to catch subtle pitch errors.
  • Your work is auto-saved every 30 seconds. Closing the tab and reopening will offer to restore.
  • To edit or delete any annotation, click Edit / Delete in the annotation list at the bottom.
  • If you re-edit error annotations after running Tap-Align, the alignment is no longer in sync. Re-run Tap-Align to refresh.
  • Hover an annotation row to preview its location on the score; click the row to seek the audio playhead there.
